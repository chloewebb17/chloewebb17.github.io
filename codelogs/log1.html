Code Log 1: HTML Semantic Tags

Semantic tags are HTML tags that clearly define their purpose to the browser and developer.

For example, a tag named "header" suggests that the text within the tag will be a header:

Non-semantic vs Semantic tags

Not all tags are semantic. If you want to make text thicker, you could use either the semantic <strong> tag, or the non-semantic <b> tag.

Strong actually indicates the function of the tag, but B is just a letter. As a designer myself, B seems a little more intuitive, but arguing about that would be... semantics (wink).

Generally, the idea is that non-semantic tags need much more context to be understood. Semantic tags describe the content within them, while non-semantic tags do not have any inherent meaning.

Let's look at some examples of both types of tags now.

Semantic Tags

  • <header>
  • <strong>
  • <section>
  • <article>
  • <figure>

Non-Semantic Tags

  • <h1>
  • <b>
  • <div>
  • <span>
  • <font>

Why use semantic tags?

There are a few good reasons why semantic tags exist.

  1. Accessibility

    2. Code is far more comprehensible to those who use screen readers, as the tags provide proper context to the code.

  2. Code maintainability

    3. Codes that use semantic tags are human-friendly and make sense to any developers who might need to edit it.

  3. Customization

    4. Semantic tags allow for CSS to be applied with far more specificity.

  4. Visibility

    5. Search engines prioritize well-structured pages that utilize semantics.
