Code Log 3: CSS Media Queries

Media queries are a function of CSS that mark a cornerstone of responsive design. They allow you to create different layouts depending on the size of the screen a webpage was viewed on. Designing responsively is a less tedious way of accounting for users viewing from differently sized screens.

Here's an overview of certain media features and their functions:

  1. Orientation

    2. This refers to whether the screen is in portrait or landscape. To be in portrait means the height is longer than the width, and the opposite is true of landscape. This can be useful for cut-and-dry situations, for example using a different photo to better fit the website depending on the orientation.

  2. Max-width

    3. This is used to define the next-smallest viewport width. It is best to use max-width in a few increments to account for different orientations such as tablets or horizontal phone screens.

  3. Aspect-ratio

    4. This often describes the relationship between the height and width of the screen. Certain aspect ratios are pretty common and it's not unwise to try and design with a specific few in mind (though it is better to imagine these things as a spectrum).

  4. Color

    5. Though we have largely written about things important to layout, you can change lots of things using media queries, such as the color of an object. Sometimes, you might want your mobile site to have a completely different design to make it more coherent and intentional. Maybe instead of white, this text box needs to be blue for the design to work. Media queries easily grant that flexibility.
